● This is a student learning project — a personal expense tracker web app called
   Spendly, built with Flask (Python). Here's what it's about:

  ---
  What it does

  Spendly is a personal finance tracker that lets users:
  - Log expenses by category, amount, date, and description
  - View spending patterns and category breakdowns
  - Filter expenses by time period (week, month, custom range)

  The UI is already designed — it has a landing page, login, and register pages
  with a clean modern look using Indian Rupee (₹) as the currency.

  ---
  Current state (starter/skeleton)

  The project is structured as a step-by-step tutorial for students to build
  out. What's already there:

  ┌────────────────┬─────────────────────────────────────────────────────────┐
  │      File      │                         Status                          │
  ├────────────────┼─────────────────────────────────────────────────────────┤
  │ app.py         │ Basic Flask routes wired up — landing, login, register  │
  │                │ work; others are stubs                                  │
  ├────────────────┼─────────────────────────────────────────────────────────┤
  │ database/db.py │ Empty — students write this in Step 1                   │
  ├────────────────┼─────────────────────────────────────────────────────────┤
  │ templates/     │ Fully built HTML (base, landing, login, register)       │
  ├────────────────┼─────────────────────────────────────────────────────────┤
  │ static/        │ CSS and JS already written                              │
  └────────────────┴─────────────────────────────────────────────────────────┘

  What students are expected to build

  The placeholder routes in app.py reveal the roadmap:
  - Step 1 — Database setup (SQLite: get_db, init_db, seed_db)
  - Step 3 — Logout functionality
  - Step 4 — Profile page
  - Step 7 — Add expense form
  - Step 8 — Edit expense
  - Step 9 — Delete expense

  So the frontend shell is given; students implement the backend logic, database
   layer, and authentication step by step.

● The error is clear — GitHub no longer accepts passwords for git operations.
  You need a Personal Access Token (PAT). Here's how to fix it:

  ---
  Step 1: Generate a PAT on GitHub

  1. Go to GitHub.com → click your profile picture (top right) → Settings
  2. Scroll down to Developer settings (bottom of left sidebar)
  3. Click Personal access tokens → Tokens (classic)
  4. Click Generate new token (classic)
  5. Give it a name (e.g. spendly-push), set expiration, and check the repo
  scope
  6. Click Generate token and copy it immediately (you won't see it again)

  ---
  Step 2: Update your remote URL to include your username

  git remote set-url origin
  https://EigenPriyam@github.com/EigenPriyam/spendly.git

  ---
  Step 3: Push again

  git push -u origin main

  When prompted:
  - Username: EigenPriyam
  - Password: paste your PAT (not your GitHub password)

  ---
  Tip: To avoid entering credentials every time, run this first so Git remembers
   them:
  git config --global credential.helper store
